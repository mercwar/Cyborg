What is Cyborg

Cyborg is a fully symbolic-opcode language and knowledge base designed for programmers, researchers, and digital architects. It combines:

  • A 256-entry opcode/symbol table mapping human-readable phrases to machine opcodes or VM bytecode.
  • MSDN-style documentation for each symbol: descriptions, C/VB6/ASM mappings, diagrams, references.
  • An interactive browser-based KB front-end (index.html + loader.js) for browsing and merging expansions.
  • ZIP-based expansion packs with seed-aware, override-capable merging logic.
  • Offline-first design with optional online expansion fetching.
  • Cross-platform compatibility for DOS, x86, Windows, and custom VMs.

Why Cyborg

  • A universal symbolic language bridging semantics and bytecode.
  • A living, extensible reference for opcodes, OS calls, and symbolic conventions.
  • A platform for VM/compiler experimentation.
  • A portable, versionable, shareable knowledge base for research and development.

How to Use (Detailed)

  1. Setup locally: Place Cyborg_KB_v1.1 anywhere on disk or cloud.
  2. Open KB: Launch index.html.
  3. Load expansions: Drop ZIP packs into expansions/ and click Load Local Expansion Pack.
  4. Online expansions: Set ONE_DRIVE_BASE in loader.js.
  5. Offline compilation: Use symbol docs + interpreter to run VM programs.
  6. Optional: Compile docs into CHM using HTML Help Workshop.
