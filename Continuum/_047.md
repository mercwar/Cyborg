    <section>
      <h2>VM Layer Stack</h2>
      <ul>
        <li>Virtual CPU (vCPU)</li>
        <li>Virtual memory (guest virtual + guest physical)</li>
        <li>Virtual devices (NIC, disk, GPU, timers, APIC)</li>
        <li>Guest OS, guest kernel, guest drivers</li>
        <li>Hypervisor control (VMX/SVM, VMCS, intercepts)</li>
        <li>Meta‑VM orchestration (multi‑VM scheduling)</li>
      </ul>
    </section>

    <section>
      <h2>Virtual CPU (Polaris)</h2>
      <ul>
        <li>vCPU registers, flags, pipeline, micro‑ops</li>
        <li>Scheduling across physical cores</li>
        <li>VM entry/exit timing mapped to Polaris</li>
      </ul>
    </section>

    <section>
      <h2>Virtual Memory (Pherkad)</h2>
      <ul>
        <li>Guest virtual → guest physical → host physical</li>
        <li>EPT/NPT tables, TLB virtualization</li>
        <li>Memory traps and access interception</li>
      </ul>
    </section>

    <section>
      <h2>Virtual Devices (Yildun)</h2>
      <ul>
        <li>Virtual NIC, disk, GPU, timers, APIC</li>
        <li>Fault zones: invalid state, timeouts, aborts</li>
        <li>VM exits on device faults</li>
      </ul>
    </section>

    <section>
      <h2>VM Control Flow (Kochab)</h2>
      <ul>
        <li>VM entry, VM exit, hypercalls</li>
        <li>Intercept routing (CPUID, I/O, MSR, CR access)</li>
        <li>Guest branching mapped to trap/return paths</li>
      </ul>
    </section>

    <section>
      <h2>Meta‑VM Layer</h2>
      <ul>
        <li>Orchestrates multiple VMs</li>
        <li>Schedules guests across hardware and time</li>
        <li>Inherits star anchors for timing, control, data, faults</li>
      </ul>
    </section>

    <section>
      <h2>Example Flow</h2>
      <p>Source:</p>
      <pre><code>vm_spawn("guest1");</code></pre>
      <p>Continuum descent:</p>
      <ul>
        <li>IR → CALL(vm_spawn, "guest1")</li>
        <li>ASM → hypercall</li>
        <li>Hypervisor → VM creation</li>
        <li>VMX → VMCS allocation</li>
        <li>Nested paging → EPT/NPT setup</li>
        <li>Virtual devices → guest hardware model</li>
        <li>Guest OS → boot sequence</li>
      </ul>
    </section>
