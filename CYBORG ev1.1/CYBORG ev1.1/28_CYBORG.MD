# **4.8.2 Diagnostic Categories**

Every IR error MUST fall into one of the following categories:

1. **TARGET_ERROR**  
2. **STRUCTURE_ERROR**  
3. **MARKER_ERROR**  
4. **EXECUTION_ERROR**  
5. **ORDER_ERROR**  
6. **BOUNDARY_ERROR**  
7. **OPCODE_ERROR**

Each category is defined below.

---

# **4.8.3 TARGET_ERROR**

Triggered when:

- no target is present  
- multiple targets exist  
- target appears after payload open  
- target appears inside payload  

### **Diagnostic Format**
```
ERROR: TARGET_ERROR ‚Äî <description>
Line <n>: "<content>"
Execution halted.
```

---

# **4.8.4 STRUCTURE_ERROR**

Triggered when:

- declarations appear inside payload  
- comments break markers  
- whitespace fragments keywords  
- unknown constructs appear  

### **Diagnostic Format**
```
ERROR: STRUCTURE_ERROR ‚Äî <description>
Line <n>: "<content>"
Execution halted.
```

---

# **4.8.5 MARKER_ERROR**

Triggered when:

- missing begin  
- missing seal  
- nested payloads  
- multiple opens  
- multiple seals  
- trailing content after seal  
- marker fragmentation  
- marker decoration  

### **Diagnostic Format**
```
ERROR: MARKER_ERROR ‚Äî <description>
Line <n>: "<content>"
Execution halted.
```

---

# **4.8.6 EXECUTION_ERROR**

Triggered when:

- verb appears outside payload  
- verb appears after seal  
- verb contains arguments  
- verb is unknown  
- verb is combined with another verb  

### **Diagnostic Format**
```
ERROR: EXECUTION_ERROR ‚Äî <description>
Line <n>: "<content>"
Execution halted.
```

---

# **4.8.7 ORDER_ERROR**

Triggered when:

- EXEC_* appears before OPEN_PAYLOAD  
- SEAL appears before EXEC_*  
- TARGET appears after OPEN_PAYLOAD  
- END_TRANSMISSION appears inside payload  

### **Diagnostic Format**
```
ERROR: ORDER_ERROR ‚Äî <description>
Line <n>: "<content>"
Execution halted.
```

---

# **4.8.8 BOUNDARY_ERROR**

Triggered when:

- payload boundaries overlap  
- payload boundaries are nested  
- payload boundaries are missing  
- content appears after seal  

### **Diagnostic Format**
```
ERROR: BOUNDARY_ERROR ‚Äî <description>
Line <n>: "<content>"
Execution halted.
```

---

# **4.8.9 OPCODE_ERROR**

Triggered when:

- IR contains unknown opcodes  
- IR contains malformed opcodes  
- IR contains illegal opcode sequences  

### **Diagnostic Format**
```
ERROR: OPCODE_ERROR ‚Äî <description>
Opcode: <opcode>
Execution halted.
```

---

# **4.8.10 Diagnostic Severity**

All IR errors are **fatal**.

There are:

- no warnings  
- no recoverable states  
- no partial validity  
- no fallback modes  

If IR is invalid, **C generation MUST NOT occur**.
