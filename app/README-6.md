
# ⭐ **README 5 — AVIS LAWBOOK**  
### *Artifact Visibility System — Full Specification & Enforcement Rules*

---

# 🔷 1. Purpose of AVIS  
AVIS (Artifact Visibility System) is the **mandatory classification and indexing standard** for all Mercwar repositories.

AVIS ensures that:

- every file identifies itself  
- every artifact is traceable  
- every schema is visible  
- every intelligence block is indexable  
- Constellation can ingest and navigate the repository deterministically  
- AIs can reason about the system without ambiguity  

AVIS is not optional.  
AVIS is the law.

---

# 🔷 2. AVIS Core Principles

## **2.1 Identity**  
Every file must declare what it is.

## **2.2 Visibility**  
Every artifact must be indexable by Constellation.

## **2.3 Determinism**  
AVIS metadata must be stable, predictable, and reversible.

## **2.4 Traceability**  
Every artifact must be traceable through ingestion, navigation, and simulation.

## **2.5 AI‑Operability**  
AVIS exists so AIs can:

- classify files  
- reason about structure  
- enforce schemas  
- validate intelligence models

# 🔷 3. AVIS Header Format

Every file must begin with:

```
AIFVS-ARTIFACT
# filename.ext
```

This header is:

- mandatory  
- non‑negotiable  
- required for ingestion  
- required for Constellation navigation

# 🔷 6. AVIS Enforcement Rules

AVIS defines strict enforcement requirements.

## **Rule 1 — Header Presence**  
Every file must contain the AVIS header.

## **Rule 2 — Filename Accuracy**  
The filename in the header must match the actual filename.

## **Rule 3 — Immutable Identity**  
Once declared, an artifact’s identity cannot change.

## **Rule 4 — Schema Compliance**  
Artifacts must follow their declared schema type.

## **Rule 5 — Constellation Visibility**  
Artifacts must be discoverable through Constellation navigation.
