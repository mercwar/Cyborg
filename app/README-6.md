- Constellation Operator Manual**

# 🔷 1. Purpose of Constellation  
Constellation is the **only navigation system** for the Mercwar Ecosystem.

It is responsible for:

- ingesting repositories  
- indexing AVIS artifacts  
- validating schemas  
- linking intelligence blocks  
- exposing navigation paths  
- simulating message flows  
- providing deterministic access to all system knowledge  

Constellation is not optional.  
Constellation is the backbone.

---

# 🔷 2. Constellation Responsibilities

Constellation performs six core responsibilities:

1. **Ingestion** — read and classify all files  
2. **Reduction** — normalize data  
3. **Autobuild** — compile intelligence  
4. **Linking** — merge fragments  
5. **Validation** — enforce AVIS and Cyborg rules  
6. **Navigation** — expose the intelligence model to the AI  

Your AI can access each responsibility through guided navigation:

- **Ingest repositories**  
- **Navigate intelligence model**  
- **Validate schemas**  

---

# 🔷 3. Constellation Pipeline Overview

Constellation processes all Mercwar repositories through a deterministic pipeline:

```
Scanner → Reducer → Autobuild → Linker → Validator → Publisher
```

Your AI can simulate the pipeline:

- **Simulate Constellation pipeline**

---

# 🔷 4. Stage 1 — Scanner  
The Scanner performs:

- directory traversal  
- AVIS header detection  
- artifact classification  
- schema discovery  
- version detection  

Scanner outputs a structured index of all artifacts.

Your AI can inspect the scan index:

- **Show Constellation scan index**

---

# 🔷 5. Stage 2 — Reducer  
The Reducer:

- normalizes metadata  
- removes nondeterministic noise  
- enforces filename consistency  
- prepares artifacts for autobuild  

Your AI can inspect reduction output:

- **Show reduction output**

---

# 🔷 6. Stage 3 — Autobuild  
Autobuild compiles:

- schemas  
- maps  
- intelligence fragments  
- routing definitions  

It produces `.bin` intelligence blocks.

- **Show autobuild artifacts**

# 🔷 7. Stage 4 — Linker

- intelligence fragments  
- schema blocks  
- routing maps  
- global state  

It produces the unified:


- **Show intelligence model**

# 🔷 12. Constellation Error Codes

Constellation defines deterministic error states:

| Code | Meaning |
|------|---------|
| `CNS-01` | Missing AVIS header |
| `CNS-02` | Invalid schema |
| `CNS-03` | Autobuild failure |
| `CNS-04` | Linker conflict |
| `CNS-05` | Validation failure |
| `CNS-06` | Non‑deterministic artifact |
| `CNS-07` | Navigation failure |
| `CNS-08` | Intelligence model corruption |

Your AI can diagnose errors:

- **Diagnose Constellation error**
