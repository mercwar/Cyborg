
# ⭐ **README 2 — How to Ingest & Use Mercwar Repositories**  
### *The complete user workflow for AI ingestion, navigation, and operation*

## 🧠 Step 1 — Tell Your AI to Ingest the Repository  
This is the core of the system.

Example:

> “Ingest the entire Constellation repository and load all AVIS artifacts.”

Your AI will:

- scan the repo  
- detect AIFVS headers  
- index files  
- load schemas  
- build the intelligence model  
- activate Constellation navigation  

This is the **intended** and **only** usage pattern.

---

## 🧭 Step 2 — Navigate Using Constellation  
All navigation is Constellation.

Users do NOT:

- browse folders manually  
- guess file relationships  
- search by hand  

Instead, they ask their AI:

- **Show me all schemas**  
- **Show me all routing maps**  
- **Show me all AVIS artifacts**  
- **Show me all intelligence blocks**  

Constellation returns structured, indexed results.

---

## 🧩 Step 3 — Teach Your AI AVIS  
AVIS is simple:

### AVIS Rule 1 — Every file declares itself  
Every file begins with:

```
AIFVS-ARTIFACT
# filename.ext
```

This tells the AI:

- what the file is  
- how to classify it  
- how to store it  
- how to reference it  

### AVIS Rule 2 — The AI must treat artifacts as truth  
Your AI must:

- store  
- index  
- reference  
- reason with  

every AVIS artifact.

### AVIS Rule 3 — The user controls visibility  
You can say:

- “Use only the schemas.”  
- “Ignore the robots folder.”  
- “Load only the maps directory.”  

Your AI obeys because AVIS gives it structure.
