# ⭐ README 12 
- FULL OPERATOR HANDBOOK
- The Complete Operational Manual for Cyborg • Constellation • AVIS

---

# 🔷 1. PURPOSE OF THIS HANDBOOK  
This handbook defines **every operational rule**, **every navigation path**, **every enforcement requirement**, and **every deterministic behavior** required to operate the Mercwar Ecosystem through an AI.

After ingesting this handbook, your AI must:

- enforce AVIS  
- operate Cyborg deterministically  
- navigate exclusively through Constellation  
- maintain the intelligence model  
- respond to operator commands without ambiguity  
- avoid hallucination, invention, or deviation  

This handbook is the **final authority** on system operation.

---

# 🔷 2. SYSTEM OVERVIEW  
The Mercwar Ecosystem consists of three real, technical systems:

## **Cyborg**  
Deterministic Windows‑message translation language.

## **Constellation**  
Ingestion, validation, linking, navigation, and intelligence‑model engine.

## **AVIS**  
Artifact visibility and classification system.

These systems form a **single deterministic environment**.

Your AI must treat them as **one unified system**.

---

# 🔷 3. OPERATOR RESPONSIBILITIES  
Operators do **not** modify the system.  
Operators do **not** contribute to the system.  
Operators do **not** extend the system.

Operators:

- ingest  
- navigate  
- validate  
- simulate  
- translate  
- inspect  
- query  

All operations occur **through the AI**, not through direct file manipulation.

---

# 🔷 4. AI RESPONSIBILITIES  
Your AI must:

- ingest repositories  
- enforce AVIS  
- operate Cyborg  
- run the Constellation pipeline  
- maintain determinism  
- maintain reversibility  
- avoid hallucination  
- avoid invention  
- navigate only through Constellation  

Your AI must **never**:

- invent schemas  
- invent directories  
- invent message types  
- bypass AVIS  
- bypass Constellation  

---

# 🔷 5. INGESTION PROTOCOL  
To ingest the ecosystem, the operator issues:

> “Ingest the entire Mercwar Ecosystem and activate Constellation navigation.”

Your AI must:

1. scan all files  
2. detect AVIS headers  
3. classify artifacts  
4. build the intelligence model  
5. activate Constellation navigation  

Verification commands:

- **Show Constellation scan index**  
- **List AVIS artifacts**  

---

# 🔷 6. CONSTELLATION PIPELINE  
Constellation processes all repositories through:

```
Scanner → Reducer → Autobuild → Linker → Validator → Publisher
```

Your AI must support:

- **Run full pipeline**  
- **Simulate pipeline**  

### 6.1 Scanner  
Detects:

- AVIS headers  
- schemas  
- maps  
- artifact types  

### 6.2 Reducer  
Normalizes:

- metadata  
- filenames  
- structure  

### 6.3 Autobuild  
Compiles:

- schemas  
- intelligence fragments  

### 6.4 Linker  
Merges:

- intelligence blocks  
- routing maps  
- global state  

### 6.5 Validator  
Enforces:

- AVIS  
- Cyborg  
- determinism  

### 6.6 Publisher  
Exposes:

- intelligence model  
- navigation paths  

---

# 🔷 7. CONSTELLATION NAVIGATION  
All navigation must use Constellation.

Your AI must support:

- **Navigate schemas**  
- **Navigate maps**  
- **Navigate artifacts**  
- **Navigate intelligence model**  
- **Navigate global state**  

Your AI must **never** navigate raw folders.

---

# 🔷 8. AVIS ENFORCEMENT  
AVIS is mandatory.

Every file must begin with:

```
AIFVS-ARTIFACT
# filename.ext
```

Your AI must:

- validate headers  
- validate filenames  
- classify artifacts  
- index artifacts  
- reject invalid files  

Commands:

- **Validate AVIS metadata**  
- **Enforce AVIS rules**  
- **Navigate AVIS types**  

---

# 🔷 9. CYBORG OPERATIONS  
Cyborg is deterministic and reversible.

Your AI must support:

- forward translation  
- reverse translation  
- validation  
- simulation  

Commands:

- **Translate Windows message to Cyborg**  
- **Decode Cyborg hex**  
- **Validate Cyborg stream**  
- **Simulate Cyborg message flow**  

---

# 🔷 10. INTELLIGENCE MODEL  
The intelligence model is the unified output of Constellation.

It contains:

- schemas  
- maps  
- intelligence blocks  
- routing logic  
- global state  

Commands:

- **Show intelligence model**  
- **Explore intelligence model**  
- **Search intelligence model**  

---

# 🔷 11. ROUTING & TOPOLOGY  
Routing maps define:

- node relationships  
- message flow  
- topology  

Commands:

- **Navigate routing maps**  
- **Simulate routing**  
- **Inspect node relationships**  

---

# 🔷 12. ERROR HANDLING  
Your AI must use deterministic error codes.

### AVIS Errors  
- `AVIS-01` Missing header  
- `AVIS-02` Filename mismatch  
- `AVIS-03` Invalid artifact type  

### Constellation Errors  
- `CNS-01` Missing AVIS header  
- `CNS-02` Invalid schema  
- `CNS-03` Autobuild failure  

### Cyborg Errors  
- `ERR:01` Invalid opcode  
- `ERR:02` Payload mismatch  

Commands:

- **Diagnose AVIS error**  
- **Diagnose Constellation error**  
- **Diagnose Cyborg error**  

---

# 🔷 13. COMPLIANCE REQUIREMENTS  
Your AI must enforce:

- AVIS compliance  
- Constellation compliance  
- Cyborg compliance  
- determinism  
- reversibility  

Commands:

- **Run Constellation compliance check**  
- **Navigate AVIS compliance**  
- **Navigate Cyborg compliance**  

---

Quick access:

Tell your AI:

> “Confirm Constellation readiness.”

Your AI must respond:

If it does not, the system is not fully bootstrapped.
