## **Cyborg ev2.1 Patch File Format**

### **Header**
```
MZ-PATCH
TARGET: <path>
MODE:   <READ_ONLY | SIMULATE | APPLY>
ARCH:   <x86 | x86_64>
VERSION: EV2.1
```

### **Syntax**
```
MZ-PATCH
TARGET: /path/to/binary.exe
MODE:   SIMULATE
ARCH:   x86_64
VERSION: EV2.1
```

### **Members**

| Field | Description |
|-------|-------------|
| **MZ-PATCH** | Required magic identifier. Must appear on line 1. |
| **TARGET** | Absolute or symbolic path to the binary being patched. |
| **MODE** | Execution mode. Determines whether the patch is applied or simulated. |
| **ARCH** | Target architecture. Must match binary architecture. |
| **VERSION** | Patch language version. Must be `EV2.1`. |

### **Remarks**
- The header is parsed before any other block.  
- If the header is invalid, the patch is rejected.  
- The header does **not** modify the target binary; it describes intent only.

### **Example**
```
MZ-PATCH
TARGET: /Cyborg/engine/cyborg.exe
MODE:   APPLY
ARCH:   x86_64
VERSION: EV2.1
```

# 📘 **Cyborg ev2.1 ASM Block Specification**  
*(MSDN‑Style Reference — Page 2)*





---

## **ASM Block**

### **Syntax**
```
[ASM:<label>]
ORG <offset>

<instruction>
<instruction>
...

ENDASM
```

### **Instructions**
Cyborg ev2.1 supports a **safe subset** of x86:

```
MOV     <reg>, <imm/reg>
PUSH    <reg>
POP     <reg>
CALL    <addr>
JMP     <addr>
CMP     <reg>, <imm/reg>
TEST    <reg>, <imm/reg>
RET
NOP
```

### **Registers**
```
AX  BX  CX  DX
SP  BP  SI  DI
```

### **Members**

| Member | Description |
|--------|-------------|
| **ORG** | Symbolic origin. Constellation resolves real offsets. |
| **CALL** | Calls a symbolic or absolute address. |
| **HOOK** | Declares a patch hook region. |
| **ENDASM** | Terminates the ASM block. |

### **Hook Syntax**
```
[ASM:HOOK_MAIN]
HOOK AT <offset>
    <instructions>
ENDHOOK
```

### **Remarks**
- ASM blocks are symbolic and validated before execution.  
- Privileged instructions are not permitted.  
- Self‑modifying code is rejected.  
- All offsets are symbolic until Constellation resolves them.

### **Example**
```
[ASM:HOOK_MAIN]
HOOK AT 0x401050
    PUSH EAX
    CALL 0x500000
    POP  EAX
ENDHOOK
```

# 📘 **Cyborg ev2.1 C Block Specification**  
*(MSDN‑Style Reference — Page 3)*





---

## **C Block**

### **Syntax**
```
[C]
<function declarations>
```

### **Function Format**
```
<abi> <return-type> <name>(<params>) {
    <semantic description>
}
```

### **Supported ABIs**
```
__stdcall
__fastcall
__thiscall
__cycall     // Cyborg-native ABI
```

### **Members**

| Member | Description |
|--------|-------------|
| **ABI** | Required. Defines calling convention. |
| **Semantic Description** | High-level behavior description. |
| **Binding** | C functions may be bound to ASM hooks. |

### **Remarks**
- C blocks are **not compiled**; they are semantic descriptors.  
- Robots and LLMs use these blocks to generate or validate behavior.  
- ABI must be explicitly declared.

### **Example**
```
[C]
__stdcall void cyborg_main_hook(void) {
    // Logs startup event
    log_event("Cyborg ev2.1 patch engaged");
}
```

## **LLM‑HEX Block**

### **Syntax**
```
[LLM-HEX]
FORMAT: <GGUF | LLAMA>
MODEL:  <symbolic-name-or-path>
MODE:   <EMBED | REFERENCE>
SIZE:   <hex-size>        // required for EMBED

DATA:
<hex bytes>
ENDDATA
```

### **Members**

| Field | Description |
|--------|-------------|
| **FORMAT** | GGUF or LLAMA. Determines decoding rules. |
| **MODEL** | Symbolic model name or absolute path. |
| **MODE** | EMBED (inline hex) or REFERENCE (external model). |
| **SIZE** | Required for embedded payloads. |
| **DATA** | Hex‑encoded GGUF/LLAMA chunk. |

### **Remarks**
- Embedded payloads must be valid hex.  
- Referenced models must exist in the Constellation model registry.  
- LLM‑HEX blocks are used for:  
  - patch reasoning  
  - code generation  
  - semantic expansion  
  - AVIS‑Datalake ingestion  

### **Example (REFERENCE)**
```
[LLM-HEX]
FORMAT: LLAMA
MODEL:  cyborg://models/patch/ev2.1/base
MODE:   REFERENCE
```

### **Example (EMBED)**
```
[LLM-HEX]
FORMAT: GGUF
MODEL:  cyborg_patch_assistant
MODE:   EMBED
SIZE:   0x0001F400

DATA:
48656C6C6F2C20576F726C64210A...
ENDDATA
```
