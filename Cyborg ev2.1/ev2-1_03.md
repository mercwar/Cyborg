### **Cyborg ev2.1 BIND Block**  
*(MSDN‑Style Reference)*

---

#### **Overview**

The **BIND** block associates **ASM hooks** with **C functions** and, optionally, **LLM‑HEX models**.  
It defines how symbolic assembly, semantic C descriptions, and LLM payloads are logically connected.

---

#### **Syntax**

```text
[BIND]
ASM:   <asm-label>
C:     <c-function-name>
LLM:   <model-symbolic-name>    // optional
MODE:  <STRICT | ADAPTIVE>      // optional
```

Multiple bindings may be declared in a single block:

```text
[BIND]
ASM:   HOOK_MAIN
C:     cyborg_main_hook
LLM:   cyborg://models/patch/ev2.1/assistant
MODE:  ADAPTIVE

ASM:   HOOK_EXIT
C:     cyborg_exit_hook
```

---

#### **Members**

| Field | Description |
|-------|-------------|
| **ASM** | Label of an ASM block or hook (e.g., `HOOK_MAIN`). Required. |
| **C** | Name of a C function declared in a `[C]` block. Required. |
| **LLM** | Symbolic name or path of an LLM‑HEX model. Optional. |
| **MODE** | Binding mode: `STRICT` (must match exactly) or `ADAPTIVE` (LLM may adjust behavior). Optional; default is `STRICT`. |

---

#### **Behavior**

- The **ASM** label must match a previously defined `[ASM:<label>]` or `HOOK` name.  
- The **C** function must exist in a `[C]` block in the same patch file.  
- If **LLM** is specified, Constellation may use the referenced model to:
  - generate code,
  - validate behavior,
  - or explain the patch.  
- **MODE** controls how strictly the binding is enforced:
  - `STRICT` → ASM and C must match declared semantics.  
  - `ADAPTIVE` → LLM may refine or extend behavior within AVIS law.

---

#### **Remarks**

- A single ASM hook may be bound to exactly one C function in a given patch.  
- Multiple ASM hooks may bind to the same C function.  
- If a BIND entry is invalid, the entire patch is rejected during validation.  
- BIND blocks are processed **after** `[ASM]`, `[C]`, and `[LLM-HEX]` blocks are parsed.

---

#### **Example**

```text
MZ-PATCH
TARGET: /Cyborg/engine/cyborg.exe
MODE:   SIMULATE
ARCH:   x86_64
VERSION: EV2.1

[ASM:HOOK_MAIN]
HOOK AT 0x401050
    PUSH EAX
    CALL 0x500000
    POP  EAX
ENDHOOK

[C]
__stdcall void cyborg_main_hook(void) {
    // Logs startup event
    log_event("Cyborg ev2.1 patch engaged");
}

[LLM-HEX]
FORMAT: LLAMA
MODEL:  cyborg://models/patch/ev2.1/assistant
MODE:   REFERENCE

[BIND]
ASM:   HOOK_MAIN
C:     cyborg_main_hook
LLM:   cyborg://models/patch/ev2.1/assistant
MODE:  ADAPTIVE
```
